# Frontend

Технологии: Vite, React 18, TypeScript, Zustand, @twa-dev/sdk, axios.

## Структура
- `src/pages` — экранные страницы, маршруты React Router (только композиция UI).
- `src/components` — переиспользуемые UI-элементы без бизнес-логики.
- `src/store` — Zustand-хранилища: состояние корзины, профиля, каталога, выбранного товара.
- `src/services` — вызовы API (axios), преобразования данных.
- `src/api` — клиент и endpoint-обертки.
- `src/types` — типы сущностей (товар, категория, пагинация).
- `src/utils` — утилиты (напр. фильтрация по поисковому запросу).
- `src/config` — конфигурация API (baseURL, заголовки, таймаут).
- `src/styles` — стили.

## Принципы
- UI = только отображение. Никаких прямых API-вызовов или бизнес-логики в компонентах/страницах.
- Состояние предсказуемо и централизовано: общие данные — в store, API — в services.
- Telegram WebApp SDK инициализируется в App, основное взаимодействие (MainButton/BackButton) привязано к страницам.

## Поток данных
1) Страница → вызывает методы store.
2) Store дергает service → service вызывает API (`api/client.ts`).
3) Ответ кладется в store, UI подписан на состояние.
